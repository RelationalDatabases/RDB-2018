The syntax of a SELECT statement that uses all possible clauses is as follows:


SELECT
  column_1, column_2,...
FROM
  table_1
  [INNER | LEFT | RIGHT] JOIN table_2 ON conditions
WHERE
  conditions
GROUP BY column_1
HAVING group_conditions
ORDER BY column_1;

SELECT statement, WHERE clause

The WHERE clause allows you to specify the search condition for the rows in the result set.

SELECT
  select_list
FROM
  table_name
WHERE
  where_condition;

The where_condition is a combination of one or more predicates using the logical operator AND, OR and NOT. A predicate in SQL is an expression that evaluates to be true, false or unkown.

Any row from table_name that causes the where_condition to evaluate to true will be included in the final result set.

Using the employees table, we want to find the employees whose job title is Sales Rep.

Try out the following query:

SELECT  
  lastname,
  firstname,
  jobtitle
FROM
  employees
WHERE
  jobtitle = 'Sales Rep';

You should see the following output in the results pane:

Even though the WHERE clause appears at the end of the SELECT statement, MySQL evaluates the expression in the WHERE clause first to select the matching rows. In this example it chooses the rows that have job title as 'Sales Rep'. MySQL then selects the columns from the select list in the SELECT clause. The highlighted (green) area contains the columns and rows in the result set.

More complex WHERE clauses.

You can form simple where conditions as above, or a very complex one that combines multiple expressions with logical operators.

For example, to find all sales reps in the office code 1 you use the following:


SELECT  
  lastname,
  firstname,
  jobtitle
FROM
  employees
WHERE
  jobtitle = 'Sales Rep' AND
  officecode = 1;

AND operator

The AND operator is a logical operator that combines two or more Boolean expressions and returns true only if both expressions evaluate to true. The AND operator returns false if either of the two expressions evaluate to false.

The following illustrates the results of the AND operator when combining, true, false and null.


        TRUE        FALSE     NULL
TRUE    TRUE        FALSE     FALSE
FALSE   FALSE       FALSE     FALSE
NULL    NULL        FALSE     NULL

SQL stops evaluating when it gets a FALSE return from a boolean expression.

The AND operator is often used in the WHERE clause of the SELECT, UPDATE, and DELETE statements.

WHERE boolean_expresssion1 AND boolean_expresssion2

We will use the customers table to demonstrate:

The following statement retrieves customers who are located in California (CA) and USA.


SELECT
  customername,
  country,
  state
FROM
  customers
WHERE
  country = 'USA' AND state = 'CA';

OR operator

The OR operator combines two boolean expressions and returns true when either boolean expression is true.

The following shows the results for the OR operator:


        TRUE        FALSE     NULL
TRUE    TRUE        TRUE     TRUE
FALSE   TRUE        FALSE    NULL
NULL    TRUE        NULL     NULL

SQL keeps evaluating the expressions until it either gets a true or gets all false, or gets a NULL. Once the result can be determined it will return a value.

Operator precedence

When you use more than one logical operator in an expression, MySQL evaluates the OR operators after the AND operators. Try out the following code:

IN operator

The IN operator allows you to determine if a specified value matches any value in a set of values.


SELECT
  column_1, column_2
FROM
  table_name
WHERE (expr|column_1) IN ('value1','value2',...);

You can use a column or an expression with the IN operator in the WHERE clause. The values in the list are separated by commas. The IN operator can also be used in the WHERE clause of other statements such as UPDATE and DELETE.

The IN operator returns 1 if the value of the column_1 or the result of the expression is equal to any value in the list, otherwise, it returns 0.

You can combine the IN operator with the NOT operator to determine if the column or expression does not match any value in the list.

We will use the offices table for demonstrate:

If you want to find the offices that are located in the U.S and France you can use the IN operator as the following query:

SELECT
  officecode,
  city,
  phone,
  country
FROM
  offices
WHERE
  country IN ('USA','France');
